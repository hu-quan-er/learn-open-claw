## 3. 读这些文件时的重点问题

建议带着这些问题读：

1. `agentCommand` 进入 `runEmbeddedAgent` 之前，已经决定了哪些默认值和策略
2. 一个 `runId` 如何和 `sessionKey/sessionId` 绑定
3. 哪些动作只在 session lane 串行，哪些动作还会再受 global lane 限流
4. runtime 原生事件是怎么被翻译成 `assistant/tool/lifecycle` 三种 stream 的
5. `NO_REPLY`、messaging tool 去重、tool summary 拼装在哪一层生效
6. compaction 触发后，retry 是怎么避免重复输出的

## 5. 当前最稳妥的一句总结

```text
OpenClaw 的 agent loop 实现，不是自己重新发明一套 agent core，
而是把一个可插拔的 agent runtime（默认内置 openclaw，亦可换 codex 等）嵌进自己的 session/queue/gateway/delivery 体系里。
```
