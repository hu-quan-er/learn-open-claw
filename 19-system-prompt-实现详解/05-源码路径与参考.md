# 源码路径与参考

这个文件列出 system prompt 专题继续深挖时最值得看的公开路径和文档。

## 1. 核心源码路径

### 1.1 `src/agents/system-prompt.ts`

最重要的文件。关注点：
- `buildAgentSystemPrompt(params)`
- `buildSkillsSection(...)`
- `buildMemorySection(...)`
- `buildProjectContextSection(...)`
- `buildHeartbeatSection(...)`
- `buildExecApprovalPromptGuidance(...)`
- `buildAssistantOutputDirectivesSection(...)`
- `buildWebchatCanvasSection(...)`
- `buildExecutionBiasSection(...)`
- `buildMessagingSection(...)`
- `buildDocsSection(...)`
- `buildAgentBootstrapSystemPromptSections(...)`
- `buildRuntimeLine(...)`

建议阅读顺序：
1. 先看 `buildAgentSystemPrompt` 参数列表，理解输入面
2. 再看 tool catalog 如何从 `toolNames` 和 `toolSummaries` 生成
3. 再看 `promptMode`、`isMinimal`、`sourceMessageToolOnly`、`sandboxInfo` 的分支
4. 最后看 stable prefix 与 dynamic suffix 的拼接位置

公开地址：
- <https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt.ts>

### 1.2 `src/agents/system-prompt-contribution.ts`

Provider prompt overlay 的类型定义。关注点：
- `ProviderSystemPromptSectionId`
- `ProviderSystemPromptContribution`
- `stablePrefix`
- `dynamicSuffix`
- `sectionOverrides`

它能解释为什么 provider 只能替换少数 section，而不是整个 prompt。

公开地址：
- <https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt-contribution.ts>

### 1.3 `src/agents/system-prompt-cache-boundary.ts`

cache boundary 的常量和辅助函数。关注点：
- `SYSTEM_PROMPT_CACHE_BOUNDARY`
- `splitSystemPromptCacheBoundary(...)`
- `prependSystemPromptAdditionAfterCacheBoundary(...)`
- `stripSystemPromptCacheBoundary(...)`

它是 stable prefix / dynamic suffix 拆分的直接实现点。

公开地址：
- <https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt-cache-boundary.ts>

### 1.4 `src/agents/system-prompt.types.ts`

prompt mode 类型定义。关注点：
- `PromptMode = "full" | "minimal" | "none"`
- `SilentReplyPromptMode = "generic" | "none"`

公开地址：
- <https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt.types.ts>

### 1.5 `src/agents/bootstrap-prompt.ts`

bootstrap pending 相关 prompt line 的构造。关注点：
- full bootstrap 怎么要求先处理 `BOOTSTRAP.md`
- limited bootstrap 怎么承认当前 run 不能完整完成
- 为什么不能假装 bootstrap 已完成

公开地址：
- <https://github.com/openclaw/openclaw/blob/main/src/agents/bootstrap-prompt.ts>

## 2. 相关源码路径

这些文件不是 system prompt 本体，但会影响 prompt 输入。

### 2.1 Skills

重点找：
- skills discovery
- eligible skills 过滤
- `formatSkillsForPrompt`
- per-agent skills allowlist
- skills prompt char limit

可能相关路径：
- `src/agents/skills.ts`
- `src/agents/skills-*`
- `src/plugins/*skill*`

### 2.2 Tool policy / schema

重点找：
- toolNames 从哪里来
- tool policy 如何 allow/deny
- provider tool schema 如何 normalize
- channel action tools 如何注入

可能相关路径：
- `src/agents/pi-tools.ts`
- `src/agents/pi-tools.policy.ts`
- `src/agents/pi-tools.schema.ts`
- `src/agents/openclaw-tools.ts`
- `src/agents/channel-tools.ts`
- `src/agents/tools/*`

### 2.3 Embedded runner / agent loop

重点找：
- `buildAgentSystemPrompt(...)` 在 agent run 中何时被调用
- `resolveAgentSystemPromptConfig(...)` 如何解析 config 旋钮（2026-06 起的中间层）
- context files 如何收集
- session manager 如何打开
- prompt 如何交给 agent runtime session（runtime 可插拔：openclaw / codex / claude-cli …）
- tool/result 如何回写 transcript

可能相关路径（**【可能已改名】**：`pi-` 前缀大概率已随 runtime 泛化去掉）：
- `src/agents/pi-embedded-runner/*`（embedded runner，对应 `runEmbeddedAgent`）
- `src/agents/pi-embedded-helpers.ts`
- `src/agents/agent-command.ts`
- `src/gateway/server-methods/agent*`

### 2.4 Context / compaction

重点找：
- system prompt + messages + tool schemas 如何共同占用 context
- `/context list` 和 `/context detail` 如何统计
- compaction 和 pruning 如何处理历史

可能相关路径：
- `src/agents/context-*`
- `src/agents/session-*`
- `src/agents/compaction-*`

## 5. 后续可继续深挖的问题

1. `before_prompt_build` hook 的 `prependContext`、`prependSystemContext`、`appendSystemContext` 最终分别落在 prompt 的什么位置？
2. provider overlay 对 GPT-5 family、Claude family、Gemini family 是否有不同默认实现？
3. `formatSkillsForPrompt` 的 budget 和排序策略如何避免热门 skill 把小众 skill 挤掉？
4. `/context detail` 统计 system prompt、tool schemas、skills list 的口径是否和 provider 实际计费完全一致？
5. channel-specific message tool hints 是由 channel plugin 直接贡献，还是由 core 汇总后统一注入？
6. sandbox path guidance 是否覆盖所有 file tool / exec tool 的路径转换边界？
7. cache boundary 在 Anthropic、OpenAI、Google、CLI transport 中分别如何落到 provider request shape？

这些问题需要继续读 OpenClaw 源码本体和 live request shaping 代码。
