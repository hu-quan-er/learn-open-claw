# 源码路径与参考

这一篇列出记忆专题继续深挖时最值得看的源码路径和公开资料。

> 提醒：当前工作区不是 OpenClaw 源码本体（`openclaw-repo/` 目前是空 checkout）。以下源码路径来自公开 GitHub 与官方文档的命名，**未在本地逐行核对**，标注的是「该去哪找」，不是「已读过」。拿到完整源码后建议据此做二次校正。

## 1. 核心源码路径

### 1.1 `src/agents/system-prompt.ts`
记忆 section 如何条件注入。关注：
- `buildMemorySection(...)` —— 何时注入、minimal 模式如何省略、依赖哪些工具是否可用
- `buildProjectContextSection(...)` —— `MEMORY.md` 等 bootstrap 文件如何进 Project Context
- bootstrap 截断逻辑（`bootstrapMaxChars` / `bootstrapTotalMaxChars` 在哪里生效）

公开地址：<https://github.com/openclaw/openclaw/blob/main/src/agents/system-prompt.ts>

（与本仓库 `19-system-prompt-实现详解/` 的 `buildMemorySection` 条目互为参照。）

### 1.2 memory 插件（`memory-core`）
`memory_search` / `memory_get` 的真正实现。重点找：
- 工具定义与参数 schema
- 混合检索权重 `vectorWeight: 0.7` / `textWeight: 0.3` 的落点
- semantic chunking（按标题切）的实现
- chunk ID `hash(source_path:start_line:end_line:content_hash:model_version)` 的生成
- top-K=3 / 200 字预览 / 按需展开
- Track A 内置 embedding（`embeddinggemma-300m-qat-Q8_0.gguf`）与 Track B（QMD）的切换

可能路径（**【推断】**）：
- `src/agents/memory/*` 或 `packages/*memory*`
- `extensions/*memory*` / `plugins/*memory*`
- 配置键入口：`agents.defaults.memorySearch.provider`

社区参照实现：memsearch（把 OpenClaw 记忆检索抽取开源）。

### 1.3 compaction / memory flush
关注：
- pre-compaction silent turn（flush）的触发点
- `reserveTokensFloor`、`softThresholdTokens`、`memoryFlush.enabled/model/systemPrompt/prompt`
- maintenance compaction vs overflow recovery 的分支
- `contextPruning.mode = "cache-ttl"` 与 cache 失效的关系

可能路径（**【推断】**）：`src/agents/compaction-*`、`src/agents/context-*`、`src/agents/session-*`

### 1.4 dreaming 子系统
关注：
- light / deep / REM 三相调度
- 六信号加权打分（relevance 0.30 / frequency 0.24 / diversity 0.15 / recency 0.15 / consolidation 0.10 / richness 0.06）
- 门槛 `minScore` / `minRecallCount` / `minUniqueQueries`
- 只有 deep 写 `MEMORY.md` 的约束
- `memory/.dreams/`、`DREAMS.md`、`memory/dreaming/<phase>/` 的写入
- 调度 `dreaming.enabled` / `frequency`（默认 `0 3 * * *`）/ `dreaming.model`

可能路径（**【推断】**）：`src/agents/dreaming/*` 或 `packages/*dream*`
